# Way of Working — CSN Chatbot
## Grupp
Orhan, Henke, Mona, Marian

## Branches
- main: endast produktionsklar kod
- dev: aktiv utvecklingsbranch
- feature/[namn]-[feature]: t.ex. feature/orhan-rag-pipeline

## Pull Requests
- Alla PRs går mot dev, inte main
- Minst 1 annan person granskar innan merge
- Ingen mergar sin egen PR

## Kommunikation
- Daglig check-in (5 min) — vad gjorde jag igår, vad gör jag idag, blockers?
- Discord/Slack för löpande kommunikation
- Möten bokas med minst några timmars varsel

## Kanban
- Issues flyttas: Todo → In Progress → In Review → Done
- Ingen tar på sig ny issue utan att stänga den gamla

## Kodstandard
- Kommentera LLM-genererad kod tydligt
- Följ DRY-principen
- Engelska variabelnamn och kommentarer i koden

## Git-commits
- Korta, tydliga commit-meddelanden på engelska
- Exempel: "add RAG pipeline for studiestod"

## Beslut
- Enkla beslut: den ansvarige bestämmer
- Större beslut: kort diskussion i gruppen, majoritet avgör
